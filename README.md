# Absolute Long System

롱 전용 통합 스캐너입니다. 

이 버전은 사용자가 기존에 정리해 온 **대통령매매법 + 길수매매법**의 장점을 하나로 묶되, 실제 배포 가능한 형태로 단순화한 **실전형 스캐너**입니다.

핵심 원칙은 업로드된 PDF의 3축입니다.

1. **파동 방향**: 상승 파동은 이전 저점을 깨지 않고, 이전 고점을 향해 간다. fileciteturn1file0
2. **RSI 기반 다이버전스**: 파동 시작/종료 힌트는 RSI에서 먼저 잡는다. fileciteturn1file0
3. **피보나치 되돌림**: 0.618~0.786은 핵심 매수 관찰 구간이며, 1 이탈은 시나리오 무효다. fileciteturn1file2turn1file4

> 중요한 점: 이것은 “절대적인 성배”가 아니라, **성배처럼 오해하기 쉬운 주관 요소를 최대한 코드화한 버전**입니다.

---

## 전략 개요

이 스캐너는 다음 조건을 합산 점수화합니다.

- **강세 RSI 다이버전스** 발생
- **3꼭지 연계 다이버전스** 우대
- 현재 가격이 **0.618~0.786 되돌림 구간**에 위치
- **Fib 1 미이탈**
- 최근 저점 구조가 **higher low** 형태로 유지
- 거래량 5봉 평균이 20봉 평균보다 증가
- 가격이 EMA20 / EMA60 위에 안착
- RSI가 과열권이 아니라 아직 눌림 구간
- 과도하게 많이 오른 종목 제거

즉, 논리는 이렇습니다.

**큰 그림상 눌림 + 구조상 higher low + RSI 다이버전스 + Fib 핵심 구간 + 거래량 재유입**

---

## 손절 / 익절 구조

이 프로젝트는 사용자의 요구대로 **퍼센트**만 보여줍니다.

- **손절**: 현재가 대비 `Fib 1(무효화 가격)`까지의 퍼센트
- **1차 익절**: 손절폭 x 1.8
- **2차 익절**: 손절폭 x 3.2

이유:
- PDF의 핵심은 **Fib 1 이탈 시 시나리오 무효**라는 점이다. fileciteturn1file4
- 따라서 절대 손절은 기술적 구조 기준으로 계산하는 것이 맞다.

## API 엔드포인트

### 1) 헬스체크

`GET /health`

### 2) 전체 롱 스캔

`GET /scan/long`

예시:

```bash
curl "http://localhost:8000/scan/long?symbol_limit=40&timeframes=1h,4h"
```

### 3) 단일 종목 스캔

`GET /scan/single`

예시:

```bash
curl "http://localhost:8000/scan/single?symbol=BTC/USDT&timeframe=1h"
```

## 한계

이 프로젝트가 강한 부분:
- PDF의 핵심 철학을 직접 반영
- 완전 자동화 가능
- 손절/익절이 퍼센트로 일관됨

이 프로젝트가 약한 부분:
- 차트 해석의 주관성을 100% 코드화할 수는 없음
- 실제 수평 저항, 뉴스, 유동성 급변, 상폐 리스크는 별도 필터 필요
- 모든 코인을 똑같이 스캔하면 노이즈가 생김

즉, **이건 완전체의 시작점이지, 시장을 무조건 이기는 절대 도구는 아니다.**
그런 도구는 현실에 없다.
